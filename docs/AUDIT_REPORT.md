# MOEX AI Hub — Сводный аудит

**Дата:** 2026-03-18
**Методология:** 5 параллельных агентов (технический аудит, продуктовый аудит, Pre Mortem, ресёрч скиллов, ресёрч Kilo Code)

---

## EXECUTIVE SUMMARY

MOEX AI Hub — перспективный продукт с хорошей базовой структурой и качественным визуальным дизайном. Однако в текущем состоянии проект **не готов к продакшен-деплою**. Обнаружено 3 критических блокера и 20+ значимых проблем.

### Критические блокеры (без исправления деплой невозможен)

| # | Проблема | Файл | Почему критично |
|---|----------|------|-----------------|
| 1 | **Tailwind загружается через CDN** | `static-site/*.html` | В air-gapped контуре CSS не загрузится — сайт будет "голым HTML" |
| 2 | **Все ссылки — плейсхолдеры** | `static-site/js/data.js` | Кнопки продуктов, гайдов, каналов не работают |
| 3 | **Зависимости без версий** | `backend/requirements.txt` | Пересборка может сломать проект |

## 1. ТЕХНИЧЕСКИЙ АУДИТ

### 1.1 Безопасность (ВЫСОКИЙ приоритет)

| Проблема | Файл | Описание |
|----------|------|----------|
| Hardcoded пароль БД | `docker-compose.prod.yml` | `POSTGRES_PASSWORD: postgres` в проде |
| Порт БД проброшен наружу | `docker-compose.prod.yml` | PostgreSQL доступен снаружи Docker |
| CORS = `*` | `backend/app/core/config.py` | Полностью отключена CORS-защита |
| correct_index утекает | `backend/app/schemas/quiz.py` | Клиент видит правильный ответ квиза |
| Нет RBAC | `backend/app/api/routers/users.py` | Любой пользователь может видеть других |
| XSS через innerHTML | `static-site/js/app.js` | Пользовательский контент вставляется без санитизации |

### 1.2 Архитектура

| Проблема | Описание |
|----------|----------|
| **Frontend и Backend не связаны** | Frontend не вызывает ни одного API-эндпоинта. Все данные в `data.js` |
| Контент захардкожен в JS | Обновление контента = пересборка Docker-образа |
| Python 3.11 (dev) vs 3.10 (prod) | Разные версии в Dockerfile и Dockerfile.prod |
| Дублирование кода | `getDisplayTags()` и `renderPromptCard()` имеют копии в разных файлах |
| README — чужой шаблон | README.md не соответствует проекту |

### 1.3 Инфраструктура

| Проблема | Описание |
|----------|----------|
| Tailwind через CDN | Гарантированно не работает в air-gapped |
| Nexus-зависимость | Dockerfile.prod зависит от `nexus-dev.tech.moex.com` |
| Нет бэкапов БД | Потеря volume = потеря данных |
| Нет health checks | Docker Compose без проверок здоровья |
| Keycloak без кэширования | Каждый запрос = HTTP-вызов к Keycloak |

## 4. ВОЗМОЖНОСТИ KILO CODE ДЛЯ HUB

### 4.1 Что Hub может распространять

| Артефакт | Формат | Air-Gapped | Приоритет |
|----------|--------|------------|-----------|
| **Custom Modes** | `.yaml` файлы | Да | Высокий |
| **Rules пакеты** | `.kilocode/rules/*.md` | Да | Высокий |
| **Starter Kit** | директория `.kilocode/` | Да | Высокий |
| **MCP конфигурации** | `.kilocode/mcp.json` | Да | Средний |
| **MCP серверы** | npm tarball / Docker | Да | Средний |
| **System prompts** | `.kilocode/system-prompt-*` | Да | Средний |
| **Memory Bank шаблоны** | `.md` | Да | Низкий |
| **.kilocodeignore** | текстовый файл | Да | Высокий |

### 4.2 Рекомендуемые Custom Modes для MOEX

1. **MOEX Security Reviewer** — только read, проверка на compliance
2. **MOEX Data Analyst** — Python/SQL, рыночные данные
3. **MOEX DevOps Engineer** — IaC, Kubernetes, ограничения на prod-конфиги
4. **MOEX Trading Systems Dev** — ASTS, SPECTRA, FIX-протоколы

### 4.3 Ключевые ограничения Kilo Code

- Локальные модели (Ollama) значительно слабее облачных
- Autocomplete привязан к Codestral, нельзя менять
- Нет git worktrees
- Custom subagents полноценно работают только в CLI
