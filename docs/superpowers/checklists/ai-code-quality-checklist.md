# Чеклист качества AI-сгенерированного кода

## Описание

Чеклист для проверки качества кода, сгенерированного AI-инструментами (Kilo Code, Cursor, GitHub Copilot, MOEX GPT). Каждый фрагмент AI-кода ОБЯЗАТЕЛЬНО проходит эту проверку перед попаданием в кодовую базу. Основан на рекомендациях OpenSSF и практиках enterprise-разработки.

---

## Принцип: AI -- джуниор-разработчик

Относитесь к AI-коду как к коду от талантливого, но неопытного разработчика: он может выглядеть чисто и правильно, но требует внимательной проверки.

---

## 1. Существование и актуальность

- [ ] **Библиотеки существуют** -- все import/require ссылаются на реальные пакеты
  - Проверить на PyPI / npm: `pip show package_name` / `npm info package_name`
  - AI может "выдумать" несуществующие пакеты (slopsquatting)
- [ ] **API и методы актуальны** -- нет deprecated функций
  - Проверить документацию: метод/функция существует в указанной версии
- [ ] **Версии совместимы** -- зависимости не конфликтуют
- [ ] **Синтаксис корректен** -- код компилируется / интерпретируется без ошибок

## 2. Безопасность

- [ ] **Нет хардкод секретов** -- пароли, ключи, токены вынесены в env
- [ ] **Входные данные валидируются** -- Pydantic, проверки типов, ограничения
- [ ] **SQL-запросы параметризованы** -- нет f-строк в запросах
- [ ] **Нет уязвимостей OWASP Top 10** -- XSS, CSRF, injection
- [ ] **Криптография актуальна** -- не используются md5, sha1 для безопасности
- [ ] **Ошибки не раскрывают внутренности** -- stack trace не попадает в ответ API

## 3. Корректность логики

- [ ] **Код решает поставленную задачу** -- не только "выглядит правильно"
- [ ] **Edge cases обработаны** -- null, пустые коллекции, граничные значения
- [ ] **Типы данных корректны** -- нет неявных приведений, потери точности
  - Особенно важно: NUMERIC для финансовых расчётов, не FLOAT
- [ ] **Асинхронность корректна** -- await на всех async-вызовах, нет блокировок
- [ ] **Нет race conditions** -- при параллельном доступе к данным

## 4. Архитектурное соответствие

- [ ] **Слой размещения правильный** -- бизнес-логика не в роутерах, SQL не в контроллерах
- [ ] **Паттерны проекта соблюдены** -- используются те же подходы, что в остальном коде
- [ ] **Нет нарушения инкапсуляции** -- приватные API не используются напрямую
- [ ] **Новые зависимости обоснованы** -- не добавлять библиотеку для одной функции

## 5. Производительность

- [ ] **Нет N+1 запросов** -- используется eager loading где нужно
- [ ] **Нет утечек памяти** -- ресурсы (файлы, соединения) закрываются
- [ ] **Алгоритмическая сложность адекватна** -- нет O(n^2) там, где можно O(n)
- [ ] **Пагинация для больших коллекций** -- не загружать все данные в память

## 6. Тестируемость

- [ ] **Код покрыт тестами** -- AI-код тестируется как минимум наравне с ручным
- [ ] **Тесты осмысленные** -- не `assert True`, а проверка реального поведения
- [ ] **Моки адекватны** -- мокируются внешние зависимости, не бизнес-логика

## 7. Лицензионная чистота

- [ ] **Нет копирования GPL-кода** -- если проект не GPL
- [ ] **Нет дословного воспроизведения** -- фрагменты из StackOverflow / GitHub (CC-BY-SA)
- [ ] **Зависимости лицензионно совместимы** -- проверить лицензии новых пакетов

## Процесс проверки

```
AI сгенерировал код
        |
        v
[1] Компилируется / запускается?  -- НЕТ --> Исправить или отклонить
        |
       ДА
        v
[2] Пройти чеклист выше           -- НЕТ --> Исправить конкретные пункты
        |
       ДА
        v
[3] Тесты проходят?              -- НЕТ --> Дебаг + исправление
        |
       ДА
        v
[4] Code review                   -- Замечания --> Итерация
        |
       ОК
        v
    Merge в кодовую базу
```

## Красные флаги AI-кода

| Сигнал | Что проверить |
|--------|--------------|
| Незнакомая библиотека | Существует ли она на PyPI/npm? |
| Слишком "умный" код | Упростить, разбить на части |
| Большие блоки без комментариев | Добавить docstrings, разобраться в логике |
| Магические числа | Вынести в константы с именами |
| Отсутствие обработки ошибок | Добавить try/except с конкретными типами |
| Код "как в туториале" | Адаптировать к архитектуре проекта |
