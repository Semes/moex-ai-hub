# Чеклист безопасности при работе с AI

## Описание

Чеклист для сотрудников Московской биржи при использовании AI-инструментов (MOEX GPT, Insight, Kilo Code, внешние LLM). Помогает избежать утечки конфиденциальных данных и нарушения политик безопасности.

---

## Перед началом работы с AI

- [ ] Убедиться, что используется одобренный корпоративный AI-инструмент (MOEX GPT / Insight)
- [ ] Проверить, что соединение защищено (HTTPS, корпоративная сеть / VPN)
- [ ] Ознакомиться с актуальной политикой использования AI в компании

## Что НЕЛЬЗЯ передавать AI

### Категория: ЗАПРЕЩЕНО

- [ ] Персональные данные сотрудников и клиентов (ФИО + паспорт, ИНН, СНИЛС)
- [ ] Пароли, API-ключи, токены доступа, сертификаты
- [ ] Данные торговых счетов и позиций реальных клиентов
- [ ] Конфиденциальные финансовые отчёты до публикации
- [ ] Исходный код критичных систем (ядро торговой системы, системы клиринга)
- [ ] Внутренние IP-адреса, схемы сети, конфигурации серверов
- [ ] Протоколы заседаний совета директоров и конфиденциальных совещаний

### Категория: ОСТОРОЖНО (только в корпоративных AI-инструментах)

- [ ] Внутренние регламенты и процедуры (без пометки "Конфиденциально")
- [ ] Код внутренних утилит и вспомогательных систем
- [ ] Обезличенные аналитические данные
- [ ] Проектная документация (без секретных данных)

## При использовании AI для кодирования

- [ ] Код не содержит хардкод credentials перед отправкой AI
- [ ] Из контекста удалены реальные connection strings и endpoints
- [ ] Тестовые данные заменены на синтетические
- [ ] AI-сгенерированный код проходит code review перед merge
- [ ] Зависимости, предложенные AI, проверены на существование и безопасность
- [ ] Лицензии предложенных библиотек совместимы с проектом

## При использовании AI для анализа данных

- [ ] Данные обезличены перед передачей AI
- [ ] Объём передаваемых данных минимально необходимый
- [ ] Результат анализа проверен экспертом перед принятием решений
- [ ] AI-анализ не используется как единственный источник для торговых решений

## При использовании AI для документов и коммуникаций

- [ ] Сгенерированный текст проверен на фактическую точность
- [ ] Нет "галлюцинаций" -- выдуманных фактов, цитат, ссылок
- [ ] Тон и стиль соответствуют корпоративным стандартам
- [ ] Документ не выдаётся за полностью авторскую работу, если существенно сгенерирован AI

## После работы с AI

- [ ] Чувствительные данные удалены из истории чата (если возможно)
- [ ] Результаты сохранены в корпоративных системах, а не только в AI-чате
- [ ] При обнаружении утечки данных -- немедленно уведомить службу ИБ

## Инциденты

При обнаружении нарушения безопасности:

1. Немедленно прекратить работу с AI-инструментом
2. Зафиксировать что именно было передано
3. Уведомить руководителя и службу информационной безопасности
4. Следовать инструкциям по реагированию на инциденты
