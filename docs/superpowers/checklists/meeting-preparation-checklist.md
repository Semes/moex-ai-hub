# Чеклист подготовки к совещанию (Dion Agent)

## Описание

Чеклист для подготовки и проведения совещаний с использованием AI-ассистента Dion. Помогает максимально эффективно использовать AI для подготовки материалов, ведения протокола и follow-up по итогам встреч.

---

## За 1-3 дня до совещания

### Определение формата

- [ ] Тип встречи определён (рабочее совещание / статус-митинг / мозговой штурм / демо)
- [ ] Продолжительность запланирована (30 / 60 / 90 мин)
- [ ] Формат участия определён (очно / онлайн / гибрид)
- [ ] Платформа выбрана (Dion / Teams / Zoom)

### Подготовка повестки

- [ ] **Цель встречи сформулирована** одним предложением
  - Пример: "Согласовать архитектуру модуля аналитики MOEX Agent Hub"
- [ ] **Вопросы повестки перечислены** (не более 5-7 пунктов)
- [ ] **Тайминг распределён** по вопросам повестки
- [ ] **Ожидаемые решения** определены для каждого вопроса
- [ ] Повестка отправлена участникам минимум за 24 часа

### Подготовка материалов (с помощью AI)

- [ ] Попросить AI подготовить:
  - [ ] Краткую сводку по обсуждаемым вопросам
  - [ ] Ключевые метрики и данные для обсуждения
  - [ ] Список рисков и открытых вопросов
  - [ ] Варианты решений для обсуждаемых проблем
- [ ] Материалы проверены на фактическую корректность (AI может "галлюцинировать")
- [ ] Презентация / документы прикреплены к приглашению

### Участники

- [ ] Список участников определён (только необходимые люди)
- [ ] Роли распределены: модератор, докладчики, ответственный за протокол
- [ ] Приглашения отправлены с повесткой и материалами
- [ ] Проверена доступность ключевых участников

## Перед началом совещания (15 мин)

- [ ] Техническая проверка: камера, микрофон, экран для демонстрации
- [ ] Dion Agent настроен для записи / транскрипции (если используется)
- [ ] Документ для заметок открыт (или AI-агент готов к ведению протокола)
- [ ] Повестка выведена на экран / в чат

## Во время совещания

### Ведение протокола (задача для AI / ответственного)

- [ ] Фиксировать:
  - [ ] Обсуждаемые вопросы и ключевые аргументы
  - [ ] Принятые решения (формулировка + кто согласовал)
  - [ ] Action items: задача + ответственный + срок
  - [ ] Открытые вопросы, требующие дополнительной проработки
  - [ ] Риски и блокеры, выявленные в ходе обсуждения

### Управление временем

- [ ] Модератор следит за таймингом по каждому вопросу
- [ ] При выходе за рамки -- зафиксировать вопрос и перенести на следующую встречу
- [ ] Последние 5 минут -- резюме и проверка action items

## После совещания (в течение 2 часов)

### Обработка протокола с AI

- [ ] Загрузить транскрипцию / заметки в AI (MOEX GPT или Dion Agent)
- [ ] Попросить AI:
  - [ ] Структурировать протокол по шаблону (см. ниже)
  - [ ] Выделить action items в формате: [Задача] -- [Ответственный] -- [Срок]
  - [ ] Сформулировать краткое резюме (3-5 предложений)
  - [ ] Подготовить письмо-рассылку участникам
- [ ] Проверить AI-сгенерированный протокол на корректность

### Рассылка и follow-up

- [ ] Протокол отправлен всем участникам
- [ ] Action items занесены в трекер задач (Jira / YouTrack)
- [ ] Следующая встреча запланирована (если нужно)
- [ ] Напоминания о дедлайнах выставлены

## Шаблон протокола совещания

```markdown
# Протокол совещания: [Название]

**Дата:** [ДД.ММ.ГГГГ]
**Время:** [ЧЧ:ММ -- ЧЧ:ММ]
**Участники:** [Список]
**Модератор:** [Имя]

## Цель встречи
[Одно предложение]

## Обсуждённые вопросы

### 1. [Вопрос]
- Обсуждение: [Ключевые аргументы]
- Решение: [Принятое решение]

### 2. [Вопрос]
- Обсуждение: [Ключевые аргументы]
- Решение: [Принятое решение]

## Action Items

| # | Задача | Ответственный | Срок |
|---|--------|---------------|------|
| 1 | [Задача] | [Имя] | [Дата] |
| 2 | [Задача] | [Имя] | [Дата] |

## Открытые вопросы
- [Вопрос, требующий проработки]

## Следующая встреча
**Дата:** [ДД.ММ.ГГГГ] **Тема:** [Тема]
```

## Промпт для AI-обработки совещания

```
Роль: Ты -- ассистент для ведения протоколов совещаний в крупной
финансовой компании (Московская биржа).

Задача: На основе предоставленных заметок / транскрипции создай
структурированный протокол совещания.

Требования:
1. Выдели ключевые решения (с указанием кто принял)
2. Сформируй список action items (задача + ответственный + срок)
3. Отметь открытые вопросы и риски
4. Подготовь краткое резюме (3-5 предложений)

Формат: Markdown по шаблону протокола MOEX

Важно: НЕ додумывай информацию, которой нет в заметках.
Если данные неполные -- отметь это явно.
```
